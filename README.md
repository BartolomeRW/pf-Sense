# Proyecto pfSense

## Descripción

Este repositorio agrupa la documentación y los entregables del proyecto educativo sobre `pfSense`. Está organizado por retos y etapas para facilitar el seguimiento, la planificación y el control de versiones.

## Objetivo

- Documentar la investigación y análisis del contexto.
- Definir el anteproyecto y la propuesta técnica.
- Registrar el control de versiones y los hitos del proyecto.
- Planificar la ejecución temporal completa.
- Reunir pruebas y entregables finales.

## Estructura del repositorio

| Carpeta | Descripción | Contenido principal |
| --- | --- | --- |
| `Reto 1 Investigación de empresa o entidad del sector en el entorno/` | Análisis del sector, empresa o entidad relacionada con el proyecto. | Investigación, contexto, fuentes y conclusiones. |
| `Reto 2 Anteproyecto/` | Definición inicial del proyecto y propuesta técnica. | Documentos del anteproyecto y estrategia. |
| `Reto 3 Control de Versiones/` | Documentación sobre el uso de control de versiones en el proyecto. | Procedimientos, commits, ramas y buenas prácticas. |
| `Reto 4 Planificación Temporal de la ejecución completa del Proyecto Inermodular/` | Cronograma y planificación de actividades. | Diagramas de Gantt, hitos y calendario. |
| `Pruebas/` | Evidencias de pruebas realizadas durante el proyecto. | Capturas, imágenes y resultados de pruebas. |
| `Documentos_Finales/` | Entregables finales del proyecto. | Versiones finales de informes y materiales de entrega. |

## Guía rápida

1. Revisa `Reto 1` para entender el marco teórico y el análisis del sector.
2. Consulta `Reto 2` para ver la propuesta inicial y el enfoque técnico.
3. Abre `Reto 3` para verificar el uso de control de versiones y el registro de cambios.
4. Usa `Reto 4` para consultar la planificación temporal y los plazos.
5. Comprueba `Pruebas` para validar resultados y evidencias.
6. Descarga los entregables definitivos desde `Documentos_Finales`.

## Buenas prácticas

- Añade referencias claras a los archivos nuevos.
